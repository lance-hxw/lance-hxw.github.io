# redis.go
封装了：
- 连接池，获取redis连接
- 锁：
	- 获取锁
		- value：生成一个uuid，如果失败就去根据时间搞个随机数
			- 这个value会被返回用于判断锁的owner
		- 用set nx设置锁，判断成功与否
	- 释放锁
		- 用value判断是不是拥有者
		- 过期时间小于5ms就不主动释放，防止边界情况
		- 流程：
			- 查询key的value值
			- 查询剩余过期时间
			- flush刷到服务器
			- 释放锁
				- 条件1：value符合
				- 条件2：过期时间大于5ms
					- 这个是为了防止快要过期了去删除
					- 结果进入条件后，到redis的请求还没执行就过期了
					- 然后被新线程获取了锁， 老线程请求过去直接把锁释放了
						- 解决：用lua原子化解决

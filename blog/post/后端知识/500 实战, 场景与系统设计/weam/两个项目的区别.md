# controller
- engine是命令模式， 做成一个注册命令的形式，init的时候注册进去
- service是直接在服务同名文件中写路由表，直接注册
	- 路由是直接用switch结合路由表
		- 这地方感觉是一个经过变动的设计，switch和路由表联合使用的，路由表现在是兜底的策略，applog等接口是没有路由表的， attention是结合，activity则完全使用路由表（似乎最新， 可能是为了weam单独开发的模块，其余部分涉及复用）
	- 路由表里的元素:
```go
type APISt struct {
	F func(c IResponsor, ctx *httpsvr.Context, et *elapsed.ElapsedTime, req interface{}) error
	P interface{}
```

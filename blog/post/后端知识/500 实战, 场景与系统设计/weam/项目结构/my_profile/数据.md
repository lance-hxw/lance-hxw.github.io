# user
- 从memberapi获取原始的用户信息
	- 请求时配置访问来源是用户自己，设置白名单和srcmid为自己mid，用于绕过隐私检查
- 更新关注粉丝等字段
	- 这里是从attentionapi拿一个返回att和fans的函数
	- 然后从中取出来赋值给app
- 转换成一个umap
- 过程中被多次访问，用于业务逻辑和数据计算
# umap
- umap被继续填充
- 最终成为响应中member_info的主要内容
成分：
- avatar_sticker
- avatar_pendant
- identity
- weam_name_alter_limit
- birth_alter_limit
- weam_name_can_alter
- birth_can_alter
- profile_update_remind: 可选，用于提示完善资料
- days：注册天数
- hots：热度值
- gods：神评数
# generalOpt

会传给getposts去拿帖子

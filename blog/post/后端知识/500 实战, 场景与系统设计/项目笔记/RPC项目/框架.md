### dispatcher 和handler
- Dispatcher
	- invocation实体(接口，方法，参数)和具体参数
		- 实现序列化
		- 扩展序列化
		- 其中接口名是String，方法名是String
		- 参数类型是Class\[], 具体参数是Object\[]
- Handlers（req， resp）
	- 反序列化得到invocation对象
	- 执行
		- 解析，得到（接口，方法，参数）以及具体参数
		- 用接口名去注册服务得到实现类
		- 在实现类中用方法名+参数类型，获取实际方法
		- 用具体参数调用方法，得到结果
	- IO回去，写到resp.getOutputStream()里
		- 注意，这里用了apache.commons-io, 的IOUils.write
		- 注意，结果是Object的话传不了，他不是serializable的，需要定义好类型
			- 然后借助ObjectOutputStream成字节流，再写
```java
public class Main {
    public static void main(String[] args) throws IOException {
        YourCustomClass obj = new YourCustomClass();
        
        // 序列化为字节流
        ByteArrayOutputStream baos = new ByteArrayOutputStream();
        ObjectOutputStream oos = new ObjectOutputStream(baos);
        oos.writeObject(obj);
        oos.close();

        // 使用 IOUtils.write 写入文件
        OutputStream outputStream = new FileOutputStream("output.dat");
        IOUtils.write(baos.toByteArray(), outputStream);
        outputStream.close();
    }
}

```

## 服务注册
- 本地注册：维护静态map，保存（interfaceName，implClass）映射
	- 这个在server本地维护就行了
- 在线注册：维护interfaceName到URL的映射
	- 这个map必须是做成注册中心，因为要server和client同时访问
### 注册中心需要为每个方法在注册的时候维护具体URL
这部分抽象成一个map，内容是
- key:String:interface+version 
- value: List\<URL>
	- 按需要实现
其需要实现功能有：
- 注册上线
- 检测下线（心跳
- 本地缓存（查询优化，不是每次都要去中心查
	- 又需要watch/订阅机制，进行同步
### 注册时需要指定版本号，同时，invocation中也要带上版本号指定调用版本
- 这里简单实现：用interfaceName+version作为key
- 复杂的，可以加上机器id等
## 代理工厂：生成目标接口的代理，实现透明化调用
### 使用jdk动态代理（面向接口）
- 使用java.lang.reflect.Proxy的newProxyInstance方法，
	- 传入 接口的类加载器，类型，和invocationhandler
	- 然后重写invoke方法作为代理逻辑
		- invoke里面用http去请求远程调用即可
		- 具体请求的地址，要去注册中心找

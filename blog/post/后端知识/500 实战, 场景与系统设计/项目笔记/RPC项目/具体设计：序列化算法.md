# JavaSerializer
基于 java io的流处理实现
即使用OOS写对象，用OISreadObject
## 优缺点：
- 简单
- 递归序列化
- 支持版本控制（serialVersionId
- 缺点：
	- 性能低
	- 体积大
	- 有安全风险
# JsonSerializer
使用jackson
用一个ObjectMapper实现
## 优缺点：
- 可读性强
- 生态丰富
- 跨语言！！
- 缺点：
	- 体积大
	- 速度差
	- 类型不安全
# XML
基于xml实现， 用J A X B， XStream等库将对象映射为XML
- 可读性，可扩展性强
- 标准化
- 缺点：
	- 体积更大了，结构臃肿
	- 性能更差了
	- 配置复杂
# Protocol Buffers（Protobuf
谷歌的高性能二进制序列化协议，需要先定义.proto生成java类
- 高性能
- 强类型
- 向前向后兼容（用字段编号）
- 跨语言
- 缺点：
	- 学习成本高
	- 不可读
	- 灵活性差，字段变动需要修改多
# Avro
hadoop常用，基于Schema，但二进制

# Thrift
facebook的RPC和序列化框架，使用IDL文件定义
相对小众

# Kryo
极高性能，专为jvm设计的高性能二进制框架
- 性能好
- 体积小
- 灵活性好
- 缺点：
	- 兼容性需要自己管理
	- 安全有问题

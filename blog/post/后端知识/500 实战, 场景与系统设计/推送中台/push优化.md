# 连接管理
- 长连接需要复用， 避免频繁建立
- 连接分片，用多个服务节点处理连接，避免单点压力过大， 可以用用户id， 地理位置等策略进行分片
# 批处理

- 对小消息，单个用户推送时可以聚合推送
- 对订阅场景， 推送时批量获取list进行处理

# 异步处理
- 用kafka等技术进行缓冲，可以用多级缓冲
- 用异步IO/协程等， 减少阻塞

# 缓存

对用户在线状态， 信息的进行缓存

对格式化消息， 缓存渲染后的模版， 减少实时计算

# 容错重试

推送失败时进行重试，高峰期可以降级/延迟推送

# 去重和幂等

防止推送两次，在业务上根据状态设计，在技术上， 基于锁和频控等机制进行

# 推送策略


根据活跃时区，兴趣，等选择合适时机， 同时做频控， 防止太多

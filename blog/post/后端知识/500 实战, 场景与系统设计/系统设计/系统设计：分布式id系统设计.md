# 需求分析
- 主键，所以必须全局唯一
- 顺序性：至少在每个表上有序，这和插入效率有关
- 长度：短的索引效率高，一般64bit就行，更短更好
- 时间顺序：
	- 保证插入顺序就能保证粗略的时间顺序，从而省去非精确需求下的时间字段
	- 但是不能完全精确，除非你变成单机
		- 这样性能太差
# 实现概述
可行的方案有：
- 多mysql自增，每台按一定step自增即可
	- 这样是无法拓展的
	- 不是全局严格递增，但是只要每个表上增就行
- 雪花算法
- 号段模式

# 存储特点

- 关注主题数据，即将需要分析的一类指标的相关数据放在一起
- 集成，需要将相关数据统一并放在一起，比如不同系统中，用户id各不相同等
- 稳定，数仓用于分析，数据相对稳定，而不是频繁变动
- 时序，数仓特别关注时间维度上的数据分析

# 分类和架构

## 实时数仓

流式处理系统，要求数据进入后，快速分析当前结果，这个查询过程应该是秒到分钟级别

主要用于，监控，实时大屏，风控，实时运营数据


主要链路：
- 数据源： mysql binlog/埋点，日志等
- 同步：kafka/pulsar
- 流处理：Flink/SparkStreaming
- 存储：clickhouse/StarRocks
- 业务


技术核心是流处理系统，基于kafka flink Clickhouse等组件

实时数仓比较复杂，需要考虑新数据的顺序，迟到和幂等等，复杂度比较高




## 离线数仓

批处理系统， T+1， 小时级出结果

主要用于 报表和历史分析

主要链路为：
- 数据来自sql/日志/saas数据等 
- 同步：cdc/datax...
- 数据湖：HDFS/S3/OSS
- 计算引擎：Hive/Spark/Trino
- 表：HiveTable/Iceberg/DeltaLake
- 调度：airflow等
- BI

即数据源头-批处理-存储-具体业务

技术核心是批处理系统，即Hive spark 等

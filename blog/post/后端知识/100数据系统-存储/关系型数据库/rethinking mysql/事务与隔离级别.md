# RU

- 读： 直接读当前数据， 有脏读
- 写：X锁
	- 只对被修改的数据加record lock

# RC

- 读：
	- 不加锁： 每次select的时候创建新的readview， 然后mvcc读
	- 加锁：record lock
- 写：X锁
	- 只对被修改的记录加record lock

# RR

- 读：
	- 不加锁：事务开始时创建readview， 然后mvcc读
	- 加锁：next-key防止幻读
- 写：
	- X锁
	- next-key
# SERIALIZABLE
- 读：加S锁（所有select都转换成lock in share mode）
- 写：X锁

# 哪些情况的读会加锁
- 显式：
	- for update ：X
	- lock in shared mode / for share：S
- 隐式（写）：
	- UPDATE语句的读
	- DELETE语句的读
	- INSERT的某些请求的
## insert操作中的锁

- Intention locks： 在执行的时候， 会在表上加意向X锁
- 在RR下， 防止幻读， 加gap lock
- 唯一性约束检查， 回对相关索引记录加S锁进行重复性检查
- 特殊操作：
	- insert...select from :会对select的表加锁
	- insert...on duplicate key update

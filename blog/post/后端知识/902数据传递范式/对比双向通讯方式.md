# websocket
[[websocket]]
全双工， 持久连接
主要优势：
- 低延迟
- 支持二进制， 文本等内容
- 协议开销小， 适合高频通讯
- 浏览器原生支持
主要缺点
- 需要长连接， 消耗资源
- 需要重连机制
- 防火墙和代理的兼容问题

# SSE
单向推送， 基于http
主要优势：
- 简单， 基于标准http
- 有自动重连
- 支持时间id和类型
- 防火墙友好
劣势：
- 只支持服务端到客户端推送
	- 反向连接需要额外接口处理
- 浏览器连接数有限制（6个）
- 只支持文本
## sse如何与post接口保持会话一致性

需要基于session/token来维护状态

# 长轮询
主要优势：
- 实时性好
- 兼容性好
- 简单
- 没有代理和防火墙问题
劣势：
- 服务器资源开销大
- 实时性还是不如ws
- 需要处理超时和重连

主要用于兼容性场景

# gRPC
基于http2， 支持多路复用
高性能， 支持二进制， 强类型， 有代码生成功能
内置负载均衡

主要缺点：浏览器不友好


# SSE和ws对比
一个双向， 一个单向+接口
ws是独立协议， 基于tcp， 通过http握手后升级到ws
sse就是http1.1的功能，基于长连接的
ws支持二进制， sse只能是文本

连接管理方面
- ws需要自己实现重连，心跳等
- sse是http的协议部分

服务器资源方面
- ws消耗较低， 建立连接后开销小
	- 适合高频通讯
- sse维持http长连接， 开销大
	- 每条消息都是完整http， 冗余多 
	- 适合低频推送

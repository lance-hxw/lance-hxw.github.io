
# 输出与收集

- 容器化场景， 使用stdout/stderr， 让日志收集器采集
	- 容器中的进程的标准输出会按配置写到指定日志文件中
	- 收集器形式：
		- sidecar，pod中专门有一个收集器，挂载日志目录进行处理
		- daemonset（k8s）， 在每个节点上运行一个日志收集器， 读取所有容器的stdout/err
		- 直接在logging driver中集成相关逻辑， 将stdout/err直接发到elk，不写日志
- 非容器化的， 写入文件， 需要配置日志滚动逻辑

# 日志等级
- debug： 只用于开发和调试记录内部状态变化
- info： 记录常规操作的日志， 如请求开始结束
- warn， 延迟， 输入验证等问题
- error， 错误
- fatal： 导致崩溃的问题
# 日志内容（应该格式化）
- 时间
- 等级
- 类/模块，方法
- 用户信息
- 请求id
- 堆栈日志
- 日志msg

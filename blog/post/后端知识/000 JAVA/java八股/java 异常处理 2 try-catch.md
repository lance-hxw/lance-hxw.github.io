## 代码中存在大量try-catch的问题

- 性能低， trycatch不会重排， 所以无法被编译器优化，效率会更低
- 代码可读性下降
- 如果catch不能正确处理，会掩盖错误
- 允许数据不一致的情况下继续执行程序，提高可用性但导致不一致扩大
- trycatch不应该作为正常逻辑的一部分， 而是只用于错误处理
- 错误处理逻辑分散，系统更复杂，应该统一捕获和处理（处理+日志）

## catch - finally执行逻辑

finally必定最后执行，如果有多个return， 就会覆盖返回值 和 其他抛出的异常


## 写循环外面还是里面

写里面： 循环继续执行， 但整体性能差
写外面：循环会终止

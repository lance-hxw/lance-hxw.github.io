# 常见场景

- 异步查询的执行者
- 定时或者批量任务
	- 如从mq接受数据乱序消费
	- 定期执行任务
# 在哪里声明与管理
- 一般在config中定义各种executor供后续使用/注入

# 实战
- config中定义一个bean
- 使用时注入， 结合CF使用
	- [[CompletableFuture]]
- 参数设置
	- io密集， 设置为n倍cpu核，或者qps\*平均异步任务数\*停留时间
	- 最大核心数根据峰值抖动估算
	- 队列：根据可接受排队秒数设置有界队列
	- 注意设置线程重命名
	- 拒绝使用快速失败（抛异常）
## 为什么不直接用CF的ForkJoinPool

全局共享， 不能隔离
别的地方性能问题可能导致其他业务问题
并行度是cpu-1
而且会丢路径追踪等
一般用于短计算任务， 而不是IO任务

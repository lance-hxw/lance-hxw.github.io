# 有哪些预设线程池

FixedThreadPool 固定线程数，无界队列
- 优点，防止过多线程
- 缺点：队列无界，可能OOM

CachedThreadPool 可缓存，线程数不固定，核心线程数0，最大线程数也无限，快速回收
- 优点：高并发快速扩展，可以执行大量短期异步任务
- 缺点：太多线程导致资源占用过大

SingleThreadExecutor 单线程
- 优点：安全，串行
- 缺点：性能差

ScheduledThreadPoll 定时任务
- 用于定时任务和周期调度

WorkSteadlingPool java 8 + ， 基于Forkjoin，提升cpu利用率
- 行为：每个线程维护自己的任务队列， 会去别的线程队列偷任务，默认线程数是cpu核心数
- 优点：性能好，多核利用率高
- 缺点：不按严格顺序执行

# 为什么要自定义线程池

- 资源可控：如FixedThreadPool无界队列，CacheThreadPool无限线程
- 隔离：不同任务定义不同线程池，使用合适参数，同时互不影响
- 可观测：可以自定义ThreadFactory，指定线程名字和优先级等
- 可以选择拒绝策略， 如抛弃最老等（默认是异常）
- 可以使用各种操作释放闲置核心线程，预热，拒绝告警等

# 如何设置参数

## 核心线程数设置
cpu：cpu cores 
IO：几倍的cpu cores， 一般是cores 乘上 （1+等待时间/计算时间）

## 具体参数

- 核心线程数：根据任务设置
- 最大线程数：是否需要弹性，如果需要，设最大线程数1.5-2，后续调整，不需要就和核心线程数一样
- 队列：必须有界
	- 处理突发的排队，按可接受时延与到达速率估算 用Aarry
		- 吞吐优先，用Linked
	- 不排队，直接同步队列+最大核心线程数
	- 优先级，使用优先队列，但任务要实现接口
- keepAliveTime 一般30s-120s，突发性或者io线程池可以更长，如要回收核心线程，可以选择配置
- threadFactory，可以配置命名规则，守护，拒绝报警等
- 拒绝策略：默认跑异常，网关层可以让提交线程执行实现背压阻塞，有时可用丢弃最老，一般不能用静默丢

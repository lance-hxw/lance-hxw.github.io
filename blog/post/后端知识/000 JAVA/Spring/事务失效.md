# Transactional何时失效
1. 如果调用方和被调用方都是一个component， 此时调用的对象不是从ioc容器中获取的代理对象， 所以没有事务
2. 被调用方不是public
3. 没开事务
4. 抛出的是检查异常， spring只处理非检查异常并处理， 若需要， 可以指定rollbackfor
5. 异常被捕获，如果是太复杂导致可能处理掉异常， 可以用非声明式事务
6. final static导致重写失败
7. 事务传播设置出错， 使用required比较好， 会将多个调用组合成一个事务
8. 不是spring管理的对象
9. 连接池+多线程， 在多个不同的数据库连接上操作， 完全无法实现事务，类似分布式事务

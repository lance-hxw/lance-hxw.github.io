# 两种形式和使用情况

## JDK 
- 必须要实现接口
- spring创建一个实现接口的代理类，方法调用委托给invocationhandler
## CGLIB
- 没有接口
- spring直接生成子类，重写相关方法
- 如果final， 就不能代理，因为需要继承

## 使用情况

spring 默认jdk，没有接口才cglib

spring boot 2.x 默认用cglib， 可以用aop.proxy-target-class 设置

# 代理失效

- 不被spring 管理
	- 比如非注入， 而是new
- 配置错误导致部分注解功能没开
- 实例调用内部方法， 没有注入代理的类
	- 比如直接this
	- 此时需要在bean中注入自己
	- 或者AopContext.currentProxy()
- 方法是private/final/static
	- 这些方法cglib 代理不了，不能继承
	- jdk代理也识别不了
		- private不可能是接口方法
		- static 对应 invokestatic， 不会被代理
		- final 在接口中也没有
- 在bean初始化阶段调用，此时代理没有创建， 还是原始对象
	- 如构造方法和PostConstruct中
